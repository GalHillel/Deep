--- commit ---
usage: deep commit [-h] [-m MESSAGE] [-a] [--ai] [-S]

Create a new commit containing the current contents of the index with a descriptive message and metadata. Using -a will automatically stage tracked file changes before committing.

options:
  -h, --help            show this help message and exit
  -m, --message MESSAGE
                        The commit message describing the changes
  -a, --all             Automatically stage modified and deleted tracked files
                        before committing (auto-stage). Does NOT include
                        untracked files.
  --ai                  Automatically generate a commit message using Deep AI
  -S, --sign            Digitally sign the commit using your identity key

Examples:
  deep commit -m "Fix bug"          # Create a commit with a manual message
  deep commit -a -m "Commit all"    # Auto-stage tracked changes and commit
  deep commit --ai -a              # AI-generated message with auto-stage
  deep commit -S -m "Signed"        # Create a cryptographically signed commit
  deep commit --allow-empty         # Create a commit even if no changes are staged

--- reset ---
usage: deep reset [-h] [--hard] [--soft] [commit]

Reset the current HEAD to a specified commit, optionally updating the index and working tree to match.

positional arguments:
  commit      The commit identifier to reset to (default: HEAD)

options:
  -h, --help  show this help message and exit
  --hard      Reset index and working tree (all local changes will be lost)
  --soft      Keep index and working tree (all changes are preserved as
              staged)

Examples:
  deep reset HEAD~1          # Undo the last commit, leaving your changes staged in the index
  deep reset --hard HEAD     # Discard all local changes (staged and unstaged) and reset to current HEAD
  deep reset --soft HEAD~2   # Move HEAD back two commits, keeping all changes in the index

Locate text patterns or regular expressions across the entire commit history and all tree objects.

positional arguments:
  query       The text string or regular expression to search for

options:
  -h, --help  show this help message and exit

Examples:
  deep search "TODO"        # Search for the literal string "TODO" in all historical versions
  deep search "^fixed:"     # Search using a regular expression

--- gc ---
usage: deep gc [-h] [--dry-run] [-v] [--prune PRUNE]

Clean up unreachable objects and optimize the internal object database to reclaim storage space.

options:
  -h, --help     show this help message and exit
  --dry-run      Display what would be cleaned up without making changes
  -v, --verbose  Display detailed information during the optimization process
  --prune PRUNE  Only prune unreachable objects older than this (seconds).
                 Default 1h.

Examples:
  deep gc                    # Execute garbage collection and database optimization
  deep gc --dry-run          # List objects that would be removed without deleting them

--- maintenance ---
usage: deep maintenance [-h] [--force]

Optimize the repository by repacking objects, updating indices, and pruning unreachable data.

options:
  -h, --help  show this help message and exit
  --force     Force run maintenance even if recently completed

Examples:
  deep maintenance           # Run scheduled maintenance tasks
  deep maintenance --force   # Force run all maintenance tasks immediately

--- debug-tree ---
usage: deep debug-tree [-h] [sha]

Forensic tool to inspect Deep tree objects. Uses repr() to reveal hidden characters for debugging purposes.

positional arguments:
  sha         The SHA-1 hash of the tree or commit object to inspect

options:
  -h, --help  show this help message and exit

Examples:
  deep debug-tree <sha>      # Reveal hidden characters in the specified tree or commit object
